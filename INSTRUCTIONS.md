## #001 — 2026-06-05 初期セットアップ

**指示内容:**
> TODOアプリをClaudeCodeを使ってAI駆動開発します。まずはClaudeCodeを使った開発を行うためのテンプレートを作ってください。TODOアプリを作るうえでは、Next.jsのみを使用し、バックエンドおよびデータベースはなし。ブラウザのデータ保存機能のいずれかを使って状態保持するようにしてください。UIライブラリは特に指定はありませんが、Tailwindを使ったデザインとヘッドレスなUIコンポーネントを組み合わせられるといいなと思います。TODOアプリはカンバンアプリ風に実装してください。ただし、チケットの状態はTODO・DOING・DONEのみで、追加や名称変更はできないようにしてください。チケットに記載できるのはシンプルなテキストのみとして、期限・担当者・詳細・備考といった属性情報は不要です。チケットの状態はドラッグアンドドロップで移動ができるようにしてください。また、今の指示も含めて、支持の履歴をテキストでワークスペースに残すようにしてください。

**実施内容:**
- Next.js 15 + TypeScript + Tailwind CSS v3 でプロジェクトを構成
- `@headlessui/react` v2 をヘッドレス UI コンポーネントとして採用
- `@dnd-kit/core` v6 をドラッグ&ドロップに採用
- localStorage による状態永続化（SSR セーフな `useLocalStorage` カスタムフック）
- カンバンボード実装: TODO / DOING / DONE の 3 カラム固定
- テキストのみのカードデータモデル（`id`, `text`, `status`, `createdAt`）
- ドラッグ&ドロップでカラム間移動（`pointerWithin` 衝突検出）
- クリックでカードのインライン編集
- `@headlessui/react Dialog` を使ったカード追加モーダル
- `CLAUDE.md` にプロジェクト設計文書を作成
- `INSTRUCTIONS.md`（本ファイル）に指示履歴を記録

**採用した主な設計判断:**
- DnD の `activationConstraint: { distance: 8 }` — クリックでの編集と誤ドラッグを両立させるため
- localStorage の hydration ガード — SSR/CSR のミスマッチを防ぐため `isHydrated` フラグを使用
- カラムを `useDroppable` の単位として定義し、カード同士の重なりを無視する単純な実装
